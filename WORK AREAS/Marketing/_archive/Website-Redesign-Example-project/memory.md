## Progress

### 2026-02-10 — Project created, wireframes started

Category: Progress

Set up the project folder and started wireframing in Framer. Decided to go with a single-page layout for the main site (homepage, services, portfolio, contact all on one scrolling page) with the blog on a separate /blog path. This keeps the user journey simple and avoids the "where do I click" problem the current site has.

### 2026-02-14 — Homepage copy first draft

Category: Progress

Drafted the homepage hero section and services overview. Used Claude to write the first draft based on my About Me files, then edited heavily. The AI draft was too formal — had to push it toward warmer, more conversational language. Took three rounds of iteration to get the tone right.

### 2026-02-14 — Tone correction logged

Category: Decision

Claude kept defaulting to corporate language ("we deliver solutions that drive results"). Told it to write like I'm explaining what we do to a friend at a coffee shop. That prompt worked much better. Added "no corporate jargon — write like a human talking to another human" to the project context for future drafts.

### 2026-02-21 — Portfolio selection finalised

Category: Decision

Narrowed the portfolio from 12 projects to 6. Criteria: recency (last 18 months), visual impact, and whether the client would give a testimonial. Dropped three projects that looked good but the clients went quiet. Better to show fewer strong examples than pad with ones we can't back up with social proof.

### 2026-03-01 — Mobile-first design decision

Category: Decision

Switched to designing mobile-first instead of desktop-first. 65% of traffic is mobile, and the current site is clearly desktop-designed-then-squeezed. Starting from mobile means the core experience is tight, and desktop just gets more breathing room. This added a day to the wireframing but will save time on responsive fixes later.

### 2026-03-01 — SEO migration plan

Category: Progress

Mapped all existing blog URLs to ensure they carry over to the new site. 23 blog posts with decent traffic. Set up 301 redirects for the 4 service pages that are changing URL structure. The /about page is being merged into the homepage, so that needs a redirect too. Documented the full redirect map in a spreadsheet.

## Lessons learned

### Copy iteration takes longer than expected

The first AI draft is never right for something this important. Budget 3-4 rounds of iteration for any customer-facing copy. The "explain it to a friend at a coffee shop" prompt was the breakthrough — worth trying on any project where tone matters.

### Start with the constraints, not the vision

The project got much clearer once we defined the hard constraints (budget, mobile, SEO, no developer). The vision is flexible — the constraints aren't. Starting with constraints narrowed the options to the ones that would actually work.
