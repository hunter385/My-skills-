# Things 3 Setup Guide

How to structure Things 3 to match your actual work: two businesses, three rhythms, one personal life.

---

## PART 1: AREAS

Areas are ongoing roles. No end date. They don't complete — they contain.

**Create these four Areas, in this order:**

1. RSG — Content
2. RSG — CEO
3. NeoWorld
4. Personal

**Why four, not two?**

RSG has two fundamentally different types of work.

Content is weekly repetition — the publishing rhythm never stops.

CEO is cycle-level strategy and leadership — bigger, slower, fewer tasks per week.

Putting them in the same bucket means your daily publishing noise drowns your strategic thinking. Separate them.

## PART 4: WHEN vs. DEADLINE

Things 3 has two date fields. Most people use them the same way. Don't.

**Deadline** = the hard date it must be done by. Use sparingly. Only for real hard stops.

**When** = the date you plan to start working on it. This is what pulls a task into your Upcoming and Today views.

**The rule:**

Set a Deadline on anything with a real hard stop (workshop day, June 30, filming day).

Set a When date earlier — 2 to 3 days out — so the task appears in your view before the deadline hits.

For repeating tasks, set a When date only. No deadline.

**Example — A-Team Workshop (due Wednesday):**

- Deadline: Wednesday
- When: Monday (shows in Upcoming two days early, gives you a buffer)

## PART 5: TAGS

Tags let you filter by context. Useful for morning setup and focus blocks.

**Create these tags:**

| Tag | Use it for |
|-----|-----------|
| #deep-work | Writing, framework creation, workshop design |
| #calls | Coaching calls, Zoom, 1:1s |
| #admin | Email, scheduling, logistics |
| #waiting | Things blocked on someone else before you can move |
| #neoworld | Cross-tag NeoWorld tasks you're personally actioning |

Don't over-tag. If everything has a tag, nothing does.

## PART 6: YOUR DAILY OPERATING SEQUENCE

### Morning Setup — 5 minutes

1. Open **Inbox** — process anything captured since yesterday
2. Open **Today** — see what's already scheduled
3. Open **Upcoming** — scan the next 7 days; pull forward anything that needs to move
4. Pick your **Daily 3** — three tasks from Today that are non-negotiable

The Daily 3 is your commitment. Everything else is bonus.

### During the Day

**Today** is your operating surface. Stay inside it.

**Anytime** view inside each Area shows tasks with no date assigned — use it when you have a free window and want to pull forward something useful.

### End of Day — 2 minutes

- Anything undone: reassign the When date or move to Someday
- Capture anything still in your head into Inbox
- Close the Today view clean

---

## PART 7: SOMEDAY

Someday is for projects and tasks you want to keep but can't action this cycle.

Use it for:
- Next-cycle projects not yet started
- Ideas you don't want to lose
- Personal projects with no timing yet

**The rule:** Don't let Someday grow past 20–25 items.

Review it at every Cooldown. Move things to active or delete them. A Someday list that never gets reviewed is just a guilt list.

---

## PART 8: INBOX

Everything you capture fast goes to Inbox first.

**Capture first. Decide later.**

Don't sort while capturing. Process Inbox once a day, during morning setup.

Three questions per item:

1. Is this a task or a project?
2. Which Area does it belong to?
3. Does it have a When date or a Deadline?

File it and move on.

---

## PART 9: WHAT THINGS 3 IS NOT FOR

Things 3 is your personal execution layer. It is not a team tool.

Don't put tasks here that belong in Basecamp or Notion. If a task requires someone else to complete it, it lives in the tool your team uses — not here.

**The clean boundary:**

- **Things 3** = what YOU need to do
- **Basecamp** = what the NeoWorld team needs to do
- **Notion** = RSG operational tracking

Crossing this line is how task systems collapse. Keep it clean.

---

## PART 10: CYCLE RESET PROTOCOL

Run this at the start of every cycle. 6 weeks for RSG. Monthly for NeoWorld.

**Five steps:**

1. **Archive completed projects** — mark done; Things 3 moves them to Logbook
2. **Review Someday** — pull anything that's now active into a new project
3. **Create next-cycle projects** — one project per cycle deliverable
4. **Confirm repeating tasks** — still current? Right frequency?
5. **Set hard dates first** — anchor the cycle with Deadlines, then set When dates working backward

Do this before the new cycle starts, not after it's already running.
